OpenPDI

OpenPDI is an unofficial, open-source effort to document and standardize police-related data from the U.S.

What we do

In May 2015, the White House launched the Police Data Initiative (PDI) in an effort to improve the relationship between citizens and police by encouraging law enforcement agencies to release data on their interactions with the communities they serve.

The Police Data Initiative promotes the use of open data to encourage joint problem solving, innovation, enhanced understanding, and accountability between communities and the law enforcement agencies that serve them.

The PDI has largely been a success, with more than 130 agencies releasing data on topics including officer-involved shootings, use-of-force incidents, and traffic stops.

However, when it comes to actually analyzing and visualizing the data, there are still a few challenges.

Data storage

The PDI provides an index of available data, but doesn't actually collect or maintain the data itself.

The agencies participating in this community of practice have chosen to release their data to the public, therefore anyone can collect the data. However, it is important to note that this initiative, the DOJ COPS Office, nor The Police Foundation are collecting this data at the national level.

This means that, in many cases, the long-term availability of data is dependent on a particular agency maintaining links to data stored on their own servers.

The OpenPDI project aims to address the potential issues with this practice by collecting and storing the data via the publicly-accessible Socrata Open Data Publisher API.

Data standardization

Since the PDI merely provides links to external data sources, there are no formatting requirements. This means that there's significant variation within the datasets submitted to the PDI (e.g., file types, column names, and value formats), which makes it difficult to perform analysis between states, agencies, and even years within the same agency.

The OpenPDI project aims to provide access to standardized PDI data via an MIT-licensed, open-source Python library.

See the GitHub repository for more information.

Data exploration

While spreadsheet-compatible file types are ideal for data analysis and research, the average person is unlikely to gain much from looking through a 5,000 line CSV file. This makes it difficult for anyone without research or programming experience to explore the available data.

OpenPDI aims to lower the entry barrier to the PDI by using openpdi.org (this website) as a means for non-technical users to explore the available data. Inspired by projects like the Knight News Challenge-funded Census Reporter, our goal is to be an impartial, easy-to-access source for data on policing in the U.S.

For example, check out our page on officer-involved shootings.

Related work

There are a number of projects that have similarities to OpenPDI, but all existing work either differs in scope (e.g., standardizing only traffic stop data) or intent. Below you'll find a summary of other notable projects.

Center for Policing Equity (CPE)

The CPE has a number of similarities with the OpenPDI project—in particular, we're both interested collecting, standardizing, and storing data on policing in the U.S. That said, there's a major difference between the two projects: data collected by the CPE isn't available to the public.

Stanford Open Policing Project

The Stanford Open Policing Project has released standardized traffic stop data for nearly every U.S. state.

The data is freely available under the Open Data Commons Attribution license.

Fatal shootings database

The Washington Post has attempted to record every fatal officer-involved shooting in the U.S. since 2015.

The data is freely available under the CC BY-NC-SA 4.0 license.
